Use Mock Servers in Testsigma to simulate backend API responses during test execution without depending on actual backend services. This article discusses creating a mock server and configuring mock rules in Testsigma.

---

> <p id="prerequisites">Prerequisites</p>
>
> Before you begin, ensure that:
> - **Testsigma Support** has enabled **Mock Servers** for your account
> - You know how to execute tests

---

## **Create a Mock Server**

1. From the left navigation bar, expand **Test Data** and select **Mock Servers**.
   ![Mock Servers](https://s3.amazonaws.com/website-static-docs.testsigma.com/new_images/projects/Updated_Doc_Images/Test_Data_Mock_Server.png)

2. You'll be redirected to the **Endpoints** page of **Mock Server Application**.
   ![Mock Server App](https://s3.amazonaws.com/website-static-docs.testsigma.com/new_images/projects/Updated_Doc_Images/Mock_Server_App_for_Testsigma.png)

3. Click **Create Endpoint**.
   ![Create Endpoint](https://s3.amazonaws.com/website-static-docs.testsigma.com/new_images/projects/Updated_Doc_Images/Create_Mock_Server.png)

4. In the **Create New Endpoint** page, enter a subdomain for the endpoint.
   ![Enter a Subdomain](https://s3.amazonaws.com/website-static-docs.testsigma.com/new_images/projects/Updated_Doc_Images/EndPoint_Name_for_Mock_Server.png)

5. Click **Create**.
   ![Create](https://s3.amazonaws.com/website-static-docs.testsigma.com/new_images/projects/Updated_Doc_Images/Create_Mock_Server_with_Domain_Name.png)
