Testsigma's Copilot is an AI-powered test-authoring and debugging tool that lets you iterate over the test case by generating, editing, and executing test steps in a single live session. You can build, run, and modify a test case during the session without exiting it.

As you work, Copilot analyzes the application screen and generates relevant test steps based on the actions you perform. Each generated step can be executed immediately, allowing you to verify whether it works before moving to the next step. This makes the test creation process more interactive and reduces the need to switch between authoring, execution, and debugging repeatedly.


Copilot also provides flexibility in how test steps are managed. You can:

   - Record actions directly from the application.
   - Set debug point or breakpoint at a step to analyse the state.
   - Insert steps at any point in the test.
   - Edit, reorder, delete, or bulk-update steps during the session.

When changes are made, Copilot automatically updates the execution flow to maintain continuity.

To help with troubleshooting, Copilot includes debugging capabilities such as Debug Points, which pause execution at selected steps, and a global debug toolbar with controls like Pause, Resume, Step Over, Skip Over and Restart. This allows you to inspect and modify test behavior without leaving the session.

Together, these capabilities make it easier to author, execute, and manage automated test cases within a single session.
